# Positioning + Messaging

Use this as source copy for `bloop.lol`, GitHub releases, and launch posts.

## Core positioning

Blooop is local voice input for terminal users who want fast dictation without cloud lock-in.

## One-line pitch options

1. Speak. Paste. Keep coding.
2. Whisper-grade dictation, fully local on your Mac.
3. The local-first voice layer for terminal workflows.

## What makes Blooop different

1. Local by default
- No server transcription calls required.
- No account, no usage meter, no “did my audio leave my machine?” ambiguity.

2. Built for coding flow
- Fast push-to-talk.
- Latch/chunk mode for longer rambles.
- Auto-paste into current app.

3. Operator-friendly
- Open implementation.
- Clear config surface.
- Local transcript history with copy/delete controls.

## Competitor framing (safe)

- Against cloud dictation tools:
  - Blooop emphasizes privacy and offline operation.
- Against polished closed desktop tools:
  - Blooop emphasizes control + transparency for technical users.
- Against one-off scripts:
  - Blooop emphasizes product-level UX (history, hotkey behavior, app mode).
